# Coze 工作流设计：「找爆款」报告生成

本文件给出在 Coze（coze.cn 国内站 / coze.com 国际站）里搭建工作流的**节点结构**和**可直接复制的提示词**。搭建好后，把「个人访问令牌(PAT)」和「工作流 ID」填到 Vercel 环境变量，前端网页就能调用它生成报告。

---

## 一、获取凭证

1. 登录 Coze → 左下角「设置」→「个人访问令牌」→ 新建令牌，复制 **PAT**。
2. 新建一个**工作流**（不是 Bot），按下方搭建，发布后在工作流详情里复制 **workflow_id**。

> 工作流 API：`POST https://api.coze.cn/v1/workflow/run`（国内站）/ `https://api.coze.com/v1/workflow/run`（国际站）。
> 服务端代理 `api/coze.js` 已封装好调用与解析，你只需配环境变量。

---

## 二、建工作流（照着点，5 步）

> 顺序：先建 4 个节点 → 再连线 → 再填提示词 → 试运行 → 发布 → 导出。

**第 1 步：新建工作流**
- Coze 左侧「工作空间」→「资源库」→「+ 资源」→ 选「工作流」→ 命名（如 `找爆款报告`）→ 进入画布。

**第 2 步：拖出 4 个节点并连线**
画布默认有「开始」「结束」两个节点，你再拖 3 个「大模型(LLM)」节点进来，按下面改名并连线（鼠标从节点右侧圆点拖到下一个节点左侧圆点）：

```
[开始]
  ├─(track,date)──▶ [LLM-A 选题分布]
  └─(track,date)──▶ [LLM-B 爆款生成]

[LLM-A 选题分布] ──(输出)──┐
                            ├─▶ [LLM-C 汇总] ──(report)──▶ [结束]
[LLM-B 爆款生成]   ──(输出)──┘
```

- **开始节点**：点开 → 添加输入参数
  - `track`：类型 `String`，必填，示例值 `自媒体AI工具`
  - `date`：类型 `String`，选填，示例值 `2026-07-09`
- **LLM-A 选题分布**（大模型节点）：
  - 模型：选一个较强的（如 `Doubao-pro` / `GPT-4o` 类）
  - 插件：开启「**联网搜索**」（让数据有真实时效，不开就偏示例）
  - 提示词：见第三节 A
  - 输出变量名设为 `topicTypes`（类型 String）
- **LLM-B 爆款生成**（大模型节点）：
  - 同样开启「联网搜索」
  - 提示词：见第三节 B
  - 输出变量名设为 `viral`（类型 String）
- **LLM-C 汇总**（大模型节点）：
  - 这个节点**不**开联网搜索
  - 在它的「输入」里引用前面两个输出：把 `topicTypes` 接到 `{{topicTypes}}`（即 LLM-A 的输出）、把 `viral` 接到 `{{viral}}`（LLM-B 的输出）
  - 提示词：见第三节 C
  - 输出变量名设为 `report`（类型 String）
- **结束节点**：输出变量选 `report`（类型 String）

**第 3 步：填提示词**
- 把第三节 A / B / C 的提示词分别复制到对应节点的「人设与回复逻辑 / 提示词」框里。注意 A、B 要选 `track`/`date` 作为输入引用（在提示词里用 `{{track}}` `{{date}}`）；C 用 `{{topicTypes}}` `{{viral}}`。

**第 4 步：试运行**
- 点画布右上「试运行」→ track 填 `自媒体AI工具` → 看结束节点是否返回一段合法 JSON。若报错，多半是某个 LLM 节点输出了多余解释文字，回去把该节点提示词末尾的「只输出 JSON，不要解释」强调一下。

**第 5 步：发布 + 拿 ID**
- 点「发布」→ 发布成功后，在工作流详情页复制 **workflow_id**（一长串字符）。这就是网页要填的环境变量之一。

> 提示：给 LLM-A / LLM-B 开启「联网搜索」插件，让数据有真实时效；否则模型只能凭训练知识生成，偏示例性质。

## 三、节点提示词（直接复制）

### LLM-A「选题分布」

```
你是一个短视频/图文平台的内容分析专家。用户会给你一个内容赛道（track）和日期（date）。
请基于你对各平台（小红书、抖音、视频号等）近期内容的了解（并尽量使用联网搜索获取最新数据），
分析该赛道下近期最热门的 5 个选题方向，并给出每个方向的流量占比（百分比，5 项加起来为 100）。

只输出如下 JSON，不要有任何解释、前言或 Markdown 代码块符号：
{
  "trackInfo": {
    "name": "<赛道名>",
    "desc": "<一句话说明该赛道追踪方向与价值，20-40字>",
    "updateDate": "<date 或今天日期>"
  },
  "topicTypes": [
    {"name": "<选题方向1>", "percent": 40},
    {"name": "<选题方向2>", "percent": 30},
    {"name": "<选题方向3>", "percent": 15},
    {"name": "<选题方向4>", "percent": 10},
    {"name": "<选题方向5>", "percent": 5}
  ]
}
要求：percent 为整数，5 项之和为 100；name 简洁（2-6字）。
```

### LLM-B「爆款生成」

```
你是一个爆款内容分析专家。用户给你赛道（track）和日期（date）。
请生成两份爆款清单（基于你对各平台近期内容的了解，尽量联网搜索获取真实趋势）：

1) newViral：近一周内流量表现最好的 10 条内容（不限粉丝量）。
2) lowFanViral：1000 粉丝以下、靠选题和内容本身起量的 10 条内容（这类对普通创作者参考价值更高）。

每条内容包含：
- format: "video" 或 "image"（视频/图文）
- title: 爆款标题
- author: 作者名（低粉请给真实感昵称）
- fans: 仅 lowFanViral 需要，字符串如 "860"（表示 860 粉）
- heat: 热度字符串如 "2.4w"
- topic: 所属选题方向（与选题分布中的 name 对应）
- breakdown: 长度为 2 的字符串数组，每条是该爆款的案例拆解（为什么爆）
- hotComments: 长度为 2 的数组，每项 {user, text, like}（like 为字符串如 "1203"），模拟评论区热议
- topicRecommend: 长度为 2-3 的字符串数组，给创作者可复用的选题推荐
- sourceUrl: 原帖链接占位，统一用 "https://www.xiaohongshu.com/explore/<id>" 形式（id 用 n1..n10 / l1..l10）
- image: 该爆款封面的图片链接（尽量用真实可访问的图床/平台图 URL；没有就留空字符串 ""，网页会自动用渐变+图标占位）

只输出如下 JSON，不要解释、不要 Markdown：
{
  "newViral": [ ...10条... ],
  "lowFanViral": [ ...10条... ]
}
```

### LLM-C「汇总成标准格式」

```
你收到两部分 JSON 字符串：A（选题分布，含 trackInfo 和 topicTypes）和 B（爆款清单，含 newViral 和 lowFanViral）。
请把它们合并成一个完整的 JSON 对象并原样输出（不要加任何解释或代码块符号）：
{
  "trackInfo": {...},
  "topicTypes": [...],
  "newViral": [...],
  "lowFanViral": [...]
}
确保是合法 JSON，字段名与上面完全一致。
```

## 四、前端如何消费

后端 `api/coze.js` 会把 Coze 返回的 `report`（JSON 字符串或对象）解析后，原样返回给前端。前端 `app.js` 期望的最终结构：

```json
{
  "trackInfo": { "name": "", "desc": "", "updateDate": "" },
  "topicTypes": [ { "name": "", "percent": 40 } ],
  "newViral": [ { "format":"video", "title":"", "author":"", "heat":"2.4w", "topic":"",
                  "breakdown":["",""], "hotComments":[{"user":"","text":"","like":""}],
                  "topicRecommend":["",""], "sourceUrl":"", "image":"" } ],
  "lowFanViral": [ { "format":"image", "title":"", "author":"", "fans":"860", "heat":"1.2w", "topic":"",
                     "breakdown":["",""], "hotComments":[...], "topicRecommend":[...], "sourceUrl":"" } ]
}
```

`id / rank / 颜色` 由前端渲染时自动补充，无需工作流输出。

## 五、导出与分享给朋友（"部署包"）

Coze 支持**工作流一键导入/导出**，这就是「把工作流部署到朋友账户」的官方方式：

1. 你按上面在自己账户建好并发布工作流。
2. 在 Coze 工作流列表里对该工作流「导出」，得到一个 `.json` 文件。
3. 把这个 `.json` 发给朋友；她在 Coze 进入「资源库 → 导入」，上传该文件并确认，工作流即被搬进她的账户（一次点击）。
4. 她用自己的 PAT + workflow_id 配置 web 应用的环境变量即可独立使用。

⚠️ 前提：导入/导出功能**仅对「个人进阶版」和「团队版」账户开放**，免费个人账户无此权限，朋友可能需要升级到进阶版（按月付费，费用不高）。
备选方案：把工作流「发布为模板」，朋友在模板中心「使用模板 / 复制」到自己的账户，限制通常更少。

> 说明：AI 无法代替你登录并部署到任何 Coze 账户——没有账户凭证，也没有可代操作的接口。最稳妥的"部署包"就是你从 Coze 自己「导出」得到的那份官方 `.json`。
